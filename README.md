# media-ip-tracker

漫画・アニメ・映画・ドラマ・小説などを、作品名検索から媒体別候補として確認し、必要な候補だけ保存して管理する個人用Webアプリです。

## 目的

v2は登録式の管理アプリではなく検索ファーストです。まず作品名で横断検索し、検索結果ページで媒体別候補・出典リンク・探すリンクを保存なしで閲覧します。保存、お気に入り、進捗状態は検索結果を見た後の補助機能として扱います。

## 公開形態

GitHub Pages で公開できるよう、アプリ本体は `/docs` 配下に配置しています。`docs/index.html` を開くとローカルでも利用できます。

### GitHub Pages 公開URL

- 公開URL: `https://bloodew418-ai.github.io/media-ip-tracker/`

### GitHub Pages 設定手順

1. GitHub のリポジトリ画面で Settings を開きます。
2. Pages を開きます。
3. Build and deployment で Source に `Deploy from a branch` を選択します。
4. Branch に `main` を選択します。
5. Folder に `/docs` を選択します。
6. Save を押します。

### 公開後の確認項目

- ページが開けること。
- トップ画面で検索欄が主導線として表示されること。
- 検索後、保存しなくても媒体別候補、出典リンク、探すリンクが表示されること。
- `localStorage` に任意保存した作品、お気に入り、媒体別進捗状態を保持できること。
- JSONエクスポート/インポートができること。
- 検索リンクが別タブで開くこと。
- 390px/360px幅で横スクロールしないこと。

## 主な機能

- 作品名による漫画・アニメ・ドラマ・映画・小説の横断検索
- 保存不要で閲覧できる検索結果ページ、目次、媒体別候補、候補なし表示
- 未保存候補への出典リンクとNetflix/Amazon等の探すリンク表示
- 検索結果カードからの任意保存
- 作品IPグループ単位のお気に入り管理
- 媒体候補単位の進捗状態管理
- 検索で見つからない場合の控えめな手動補助保存
- localStorage保存
- JSONエクスポート/インポート

## データ方針

外部サービスの情報は、以下を分けて扱います。

1. 公開APIから取得した検索候補
2. 出典リンクとして案内する情報
3. 探すリンクとして案内する外部検索URL
4. ユーザーが任意保存または手動補助保存した情報

配信中・販売中は原則として断定せず、検索候補、出典リンク、探すリンク、保存済み進捗状態を分けて表示します。
